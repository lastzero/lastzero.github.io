Validation in the MVC design pattern

Jan 5, 2009

Hello Martin, your article on Notification implies that validation should be done in the Model layer. There are numerous voices that see the need for validation in the Controller, as it receives user input in the first place. I argue that actually both should be done in Web applications: – First you want to make sure that data integrity is given. You should not outsource this essential task to the Controller or the View.
